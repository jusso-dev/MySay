# Security policy

## Supported versions

MySay is in early development. Security and privacy fixes are made on the
latest `main` branch; older commits and forks are not supported by the project.

## Reporting a vulnerability

Please do not disclose a vulnerability in a public issue, discussion, or pull
request. Use GitHub's **Report a vulnerability** form in the repository's
Security tab. If private vulnerability reporting is unavailable, contact
`@jusso-dev` privately through the contact methods on the maintainer's GitHub
profile.

Include the affected version or commit, reproduction steps, impact, and any
suggested mitigation. Do not include real children's names, photos, recordings,
health information, or exported MySay libraries. A minimal synthetic test case
is preferred.

You can expect acknowledgement within seven days. The maintainer will validate
the report, coordinate a fix and disclosure timeline where appropriate, and
credit reporters who wish to be named.
